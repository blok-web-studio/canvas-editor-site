Why canvas

The web's document problem.

Contenteditable editors rely on the browser to render text. Every browser does it differently. Page breaks, tables, and line spacing shift depending on where you open the document. Canvas-editor draws every pixel itself — the output is identical everywhere.

Features

Everything a document needs.

Pagination, tables, math, forms, plugins — not bolted on. First-class.

Native Pagination

Document-style page breaks with headers, footers, and watermarks. Content flows across pages like Word.

LaTeX Math

Inline and block equations rendered natively. No iframes, no external renderers.

Form Controls

Checkboxes, radio buttons, dropdowns, date pickers, text inputs — all interactive and data-bound.

Rich Tables

Cell merge, resize, column width, styled headers. Tables that render identically everywhere.

Plugin System

Extend the editor with custom content types, toolbar buttons, and behaviors. Headless by design.

Print & PDF

What you see is what you print. Pixel-perfect PDF export with proper margins and page numbers.

Quick Start

Integrate in minutes.

One npm package. A few lines of code. Your documents rendered on canvas.

Terminal
$ npm install @hufe921/canvas-editor
import { Editor } from '@hufe921/canvas-editor' const editor = new Editor({ container: document.getElementById('editor'), data: /* your document data */, mode: 'edit', // or 'read' }) editor.destroy()
